# 1
Through the visualization of the asthma dataset, I learned that childhood asthma prevalence fluctuated through the early 2000s, peaked around 2008 to 2010, and then declined toward 2017 and started grow again. Poverty acts as a strong predictor, as children living below the federal poverty line consistently show higher asthma rates than those in higher income brackets. An unexpected finding is that uninsured children actually show lower reported asthma prevalence than insured children. Rather than indicating better health, this paradox most likely points to underdiagnosis, as children without insurance have less access to the clinical evaluations needed to be formally diagnosed with asthma.
# 2
I created the interface using R Shiny with the bslib package to establish a modern, two-tab structure. Data preparation was a crucial first step, where I filtered out missing estimates, isolated only asthma-related data using string matching, renamed verbose panel descriptions to short labels like Current Asthma, extracted numeric start years, and applied custom factor levels so demographic groups appeared in a logical rather than alphabetical order. The style customizations and layout were guided by the goal of creating a clean, academic appearance. I used a consistent sidebar layout on both tabs to reduce cognitive load, applied a flatly theme with a custom blue brand color, utilized colorblind-safe palettes from RColorBrewer, and chose ridge plots with jittered points alongside line plots with standard error ribbons to communicate data distribution without visual clutter.
# 3
The reactive graph structure of the application operates in three isolated branches to maximize efficiency. First, the data loading happens only once at the session start, and an observer dynamically updates the user interface checkboxes based on the available conditions. The first reactive expression, handling the trend data, listens only to the demographic and condition inputs from the first tab to filter and format the dataset for the line plot. The second reactive expression, handling the facetted ridge plot, listens exclusively to the demographic input on the second tab to prepare data for the density plot. This separated structure ensures that when a user interacts with one tab, it does not trigger unnecessary recalculations in the other.
